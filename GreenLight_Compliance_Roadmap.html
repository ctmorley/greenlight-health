SECTION 01

Executive Summary & Current State Assessment

GreenLight has invested heavily in security engineering from Sprint 13. The compliance gap is predominantly in policy documentation, process formalization, and third-party assessments — not in technical controls.
The GreenLight advantage: Most healthcare startups begin their SOC 2 journey at 0% technical control implementation. GreenLight has completed 70%+ of the technical security controls through Sprint 13’s security hardening sprint. The remaining work is primarily organizational: writing policies, implementing endpoint management, conducting training, and engaging auditors. This dramatically compresses the timeline and reduces remediation costs.

e1 vs. r2 strategy: The HITRUST e1 assessment (44 controls) is the fastest path to certification and sufficient for most initial enterprise sales. The r2 assessment adds risk-based controls (190+ total) and is the “gold standard” that large IDNs and national payers require. Recommendation: achieve e1 first (Month 9–12), then pursue r2 (Month 12–18) while revenue justifies the investment.

Certification Pathway & Timeline

A parallel-track approach targeting SOC 2 Type II in 9–12 months and HITRUST r2 in 18 months. The observation periods overlap to maximize velocity.

Months 1–2: Foundation & Setup

  • Select and deploy compliance platform (Vanta/Drata/Secureframe)
  • Write core security policies (Information Security, Access Control, Change Mgmt)
  • Begin automated evidence collection
  • Engage SOC 2 audit firm and HITRUST assessor
  • Deploy endpoint management solution
Milestone: Compliance platform live, 5+ policies approved

Months 2–3: Gap Remediation

  • Deploy MFA enforcement across all systems
  • Complete employee security awareness training
  • Schedule and execute first penetration test
  • Implement continuous vulnerability scanning
  • Conduct formal risk assessment
  • Establish vendor risk assessment program
Milestone: All critical gaps remediated

Month 3: SOC 2 Type I Audit

  • Point-in-time assessment of control design
  • Auditor evaluates all documented controls
  • Produces Type I report (design effectiveness)
  • Identifies any remaining remediation items
Milestone: SOC 2 Type I report issued

Months 3–9: SOC 2 Type II Observation

  • 6-month observation period begins
  • Continuous evidence collection via compliance platform
  • Quarterly access reviews executed
  • Ongoing vulnerability scanning and remediation
  • Incident response tabletop exercises
Milestone: Observation period complete

Months 6–9: HITRUST e1 Self-Assessment

  • Map 44 essential controls to evidence
  • Complete HITRUST MyCSF portal entries
  • Internal readiness review
  • Submit for HITRUST e1 validated assessment
Milestone: HITRUST e1 assessment submitted

Months 9–12: Certifications Issued

  • Month 9: SOC 2 Type II report issued
  • Month 9–12: HITRUST e1 certification issued
  • Begin HITRUST r2 scoping (190+ controls)
Milestone: SOC 2 Type II + HITRUST e1 certified

Months 12–18: HITRUST r2 Full Assessment

  • Complete risk-based control implementation

  • HITRUST r2 certification issued

SECTION 05

Technical Requirements — What to Build

Remaining technical work to close compliance gaps. Organized by priority and effort. Most items are configuration/deployment rather than new application code.

P0: MFA Enforcement

Enforce multi-factor authentication across all systems: production infrastructure, source code (GitHub), compliance platform, cloud consoles (Vercel, Neon). Use Azure AD / Entra ID or Okta as identity provider.

Effort: 1–2 days • Blocks: SOC 2 CC6, HITRUST 11, 12

Critical Week 1

P0: Endpoint Management (MDM)

Deploy MDM solution (Jamf for macOS, Kandji, or Microsoft Intune) on all developer and employee devices. Enforce disk encryption, screen lock, OS patching, and remote wipe capability.

Effort: 1–2 weeks • Blocks: HITRUST 03, 05

Critical Week 1–2

P1: Continuous Vulnerability Scanning

Implement automated vulnerability scanning beyond CI/CD. Options: Qualys, Tenable, or Snyk for application-level scanning. Run scheduled scans weekly with automated ticket creation for findings.

Effort: 2–3 days • Blocks: SOC 2 CC7, HITRUST 08

High Week 2–3

P1: Monitoring & Alerting Rules

Configure alert rules for security events: failed login attempts (>5 in 10 min), unauthorized PHI access attempts, bulk data export, elevated privilege usage, after-hours access patterns. Route to PagerDuty or Opsgenie.

Effort: 3–5 days • Blocks: SOC 2 CC4, CC7, HITRUST 13

High Week 2–4

P1: Database Audit Logging Enhancement

Enable comprehensive database audit logging on Neon PostgreSQL. Track all DDL changes, privileged operations, and schema modifications. Ensure log retention meets SOC 2 requirements (minimum 1 year).

Effort: 1–2 days • Blocks: SOC 2 CC4, HITRUST 13

Medium Week 3–4

P1: Backup Verification Testing

Implement automated backup restoration testing. Verify Neon database backups can be restored to a point-in-time. Document RPO (Recovery Point Objective) and RTO (Recovery Time Objective). Run monthly.

Effort: 2–3 days • Blocks: SOC 2 A1, HITRUST 07

Medium Week 3–4

P2: Container & Dependency Scanning in CI/CD

Add container image scanning (Trivy or Snyk Container) to the existing CI/CD pipeline. Scan for known CVEs in base images and dependencies. Block deployments with critical/high findings.

Effort: 1 day • Blocks: SOC 2 CC8, HITRUST 07, 08

Enhancement Week 4–6

P2: Network Segmentation Review

Document and verify network segmentation between application tiers. Ensure database is not directly accessible from the internet. Review Vercel/Neon network isolation. Document firewall rules and access paths.

Effort: 1–2 days • Blocks: SOC 2 CC6, HITRUST 09

Enhancement Week 4–6
Total estimated technical effort: 4–6 weeks of engineering time, much of it parallelizable. The largest items (MDM deployment, vulnerability scanning, monitoring rules) are configuration work, not new application development. No changes to the GreenLight core application codebase are required — these are infrastructure and tooling items.

Compliance platform advantage: Vanta, Drata, and Secureframe all provide pre-written, attorney-reviewed policy templates customized for healthcare SaaS companies.
